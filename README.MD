# 🚀 快速入门

## 系统要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | Windows 10 64-bit |
| 内存 | 4GB 以上 |
| Node.js | v18 或更高 |
| Git | 任意版本 |
| 网络 | 首次启动需联网下载依赖 |

## 首次启动

1. **双击** `OpenClaw Dashboard` 图标启动应用
2. 系统自动进入**环境检测**页面，逐一检查：
   - 🖥️ Windows 版本和 DirectX 11 支持
   - 💻 PowerShell 可用性和版本
   - 📦 Node.js 是否安装及版本号
   - 🔧 Git 是否可用
   - 🐍 Python 是否可用（可选组件）
   - 🔗 OpenClaw CLI 是否已安装
   - ⚙️ Gateway 配置文件是否存在且有效
3. 根据检测结果采取不同操作：
   - ✅ **全部通过** → 30 秒后自动进入主界面，或点击「进入APP」直接进入
   - ⚠️ **有警告项** → 可选择修复后进入，也可忽略直接进入
   - ❌ **致命错误** → 必须修复才能启动，点击「🔧 修复」按钮自动处理
4. 进入主界面后，点击顶部状态栏的 **▶ 启动** 按钮启动 Gateway 服务

## Gateway 管理

Gateway 是 OpenClaw 的核心服务进程，管理所有 AI 会话、渠道连接和模型调用。

| 操作 | 按钮 | 详细说明 |
|------|------|----------|
| 启动 | 「▶ 启动」 | 自动执行环境检测，通过后启动 Gateway 进程 |
| 停止 | 「■ 停止」 | 优雅停止 Gateway，断开所有 WebSocket 连接 |
| 重启 | 「🔄 重启」 | 自动执行 停止 → 等待 → 启动 流程 |
| 自启监控 | 「服务自启」复选框 | Gateway 异常退出时每 15 秒自动检测并重启 |

### 状态栏指示灯

| 状态显示 | 含义 | 需要操作 |
|----------|------|----------|
| 🟢 Gateway [已连接] | Gateway 正常运行，WS 连接成功 | 无需操作 |
| 🔴 Gateway [未连接] | Gateway 未启动或连接断开 | 点击「▶ 启动」 |
| 🟢 Session [在线] | 会话通道已建立，可正常对话 | 无需操作 |
| 🔴 Session [离线] | 会话通道未建立 | 等待 Gateway 完全启动 |
| 📊 Agents: N | 当前活跃 Agent 数量 | 参考信息 |
| 📊 Sessions: N | 当前活跃会话数 | 参考信息 |
| ⏱️ WS延迟 Nms | WebSocket 延迟 | >1000ms 表示网络不佳 |

# 🖥️ 控制台

控制台提供完整的 PowerShell 终端集成，可以直接在 Dashboard 中执行系统命令和查看 Gateway 日志。

## 终端功能

- ✅ **实时日志流** — Gateway 启动后的 stdout/stderr 输出实时显示
- ✅ **命令交互** — 支持直接输入和执行 PowerShell 命令
- ✅ **Conda 支持** — 勾选后自动激活 Conda base 环境
- ✅ **UTF-8 中文** — 自动设置编码，完美支持中文输出
- ✅ **彩色输出** — 命令行绿色高亮，标准输出白色，错误信息红色

## 命令输入

在终端下方的输入框中输入命令，按 **Enter** 或点击 **执行** 按钮运行：

```powershell
# 常用命令示例
openclaw status          # 查看 OpenClaw 运行状态
openclaw doctor          # 运行系统诊断
openclaw doctor --fix    # 自动修复检测到的问题
openclaw gateway start   # 命令行启动 Gateway
openclaw gateway stop    # 命令行停止 Gateway
openclaw models list     # 列出所有可用模型
openclaw sessions list   # 列出所有活跃会话
dir                      # 列出当前目录文件
echo 你好                # 测试中文输出
```

> 💡 输入框执行的命令会在**独立结果显示区**中显示，不会干扰上方的终端日志流。

## Conda 环境切换

输入框左侧有 **☐ Conda** 复选框：

| 状态 | 行为 | 示例 |
|------|------|------|
| ☐ 未勾选（默认） | 命令直接通过 PowerShell 执行 | `python script.py` |
| ☑ 已勾选 | 通过 `conda run -n base` 执行 | 自动激活 base 环境 |

> 💡 如需使用非 base 环境，不勾选复选框，手动写：`conda activate myenv && python script.py`

## 结果显示区

- 📋 **可选中复制** — 鼠标选择文字后 Ctrl+C
- ✕ **可关闭** — 点击右上角 ✕ 按钮
- 🎨 **彩色显示** — 绿色=命令行，白色=标准输出，红色=错误输出

## 快捷操作按钮

| 按钮 | 功能 | 详细说明 |
|------|------|----------|
| 📁 自定义目录 | 创建 Junction 链接 | 将 `~/.openclaw` 重定向到自定义路径 |
| 🔐 添加 WS 权限 | 补全 WS 配置 | 自动添加 `gateway.controlUi.allowedOrigins` |
| 🔓 启用 WS 验证 | 开启 WS 认证 | 设置 `allowInsecureAuth` |
| 🔌 WS 测试 | 测试连接 | 验证 WebSocket 连接并显示延迟 |

# 🤖 模型管理

管理所有 AI 模型和 Provider（服务商），这是 Dashboard 最核心的功能之一。

## Provider 概念

Provider 是 AI 模型的服务商。每个 Provider 包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | Provider 唯一标识 | `openrouter`、`nvidia` |
| Base URL | API 接口地址 | `https://openrouter.ai/api/v1` |
| API Key | 访问密钥 | `sk-or-v1-xxx` |
| API 类型 | 接口协议 | `openai-completions` |
| Models | 该 Provider 下的模型列表 | `[gpt-4o, claude-3]` |

## 添加模型

### 方式一：快速添加（推荐）

1. 点击「＋ 添加模型」打开弹窗
2. 展开「⚡ 热门模型快速添加」区域
3. 浏览预设模型列表，找到想要的模型
4. 点击「+ 添加」按钮
5. 如果是新 Provider，会弹出输入框要求填写 API Key
6. 模型自动添加，同时同步到 Agent 白名单

预设模型包含 NVIDIA、OpenRouter 等平台的热门免费模型，开箱即用。

### 方式二：手动添加

1. 点击「＋ 添加模型」打开弹窗
2. 选择 Provider：
   - 从下拉列表选择**已有 Provider**
   - 或输入**新 Provider 名称**（需同时填写 Base URL、API Key、API 类型）
3. 填写模型信息：
   - **Model ID** — 模型标识符，如 `gpt-4o`、`claude-sonnet-4`
   - **Model Name** — 显示名称，如 `GPT-4o`（可选，默认使用 Model ID）
4. 点击「🧪 测试」验证模型是否可用
5. 确认添加

### 重复检测机制

系统提供 3 层防重复保护：

| 层级 | 触发时机 | 表现 |
|------|----------|------|
| 实时提示 | 输入 Model ID 时 | 输入框下方显示橙色 ⚠️ 警告文字 |
| 提交检查 | 点击「✅ 添加」时 | 弹窗提示，阻止添加 |
| 后端兜底 | IPC 调用时 | 返回错误信息，阻止写入配置 |

当选择已有 Provider 后输入已存在的 Model ID，输入框下方会显示：`⚠️ 模型「xxx」已存在于「provider」中`

## 设置主模型

每个模型卡片上有操作按钮：

| 按钮 | 作用 | 说明 |
|------|------|------|
| 📝 主模型 | 设为文本对话主模型 | 所有新会话默认使用此模型 |
| 🖼️ 主模型 | 设为图像/多模态主模型 | 图片分析、视觉任务使用 |
| 📝 +备用 | 添加到文本备用列表 | 主模型不可用时自动切换 |
| 🖼️ +备用 | 添加到多模态备用列表 | 多模态主模型的后备 |

**自动故障转移**：当主模型请求失败（超时、限流、余额不足等），系统自动尝试备用模型列表中的下一个，无需手动干预。

# 💬 会话管理

管理所有 AI 对话会话，支持多会话并行。

## 会话列表

左侧显示所有活跃会话，按最近更新时间排序：

- 📌 **主会话** — 标记为主会话，不可删除
- 🔍 **搜索过滤** — 输入关键词过滤会话名称
- 📊 **会话信息** — 显示渠道、最后活跃时间
- 🤖 **模型显示** — 显示当前会话使用的模型名称

## 发送消息

1. 从左侧列表**点击选择**一个会话
2. 底部输入框自动获得焦点
3. 输入消息内容
4. 按 **Ctrl+Enter** 或点击「发送」按钮
5. AI 回复以流式输出实时显示

### 输入方式

| 操作 | 按键 | 说明 |
|------|------|------|
| 发送消息 | Enter 或 Ctrl+Enter | 直接发送 |
| 换行 | Shift+Enter | 多行输入 |
| 中断生成 | Esc 或点击「停止」 | 终止 AI 输出 |

### 流式输出特性

AI 回复采用流式输出（打字机效果）：

- 💭 **思考过程** — 点击「显示思考过程」可查看 AI 的推理步骤
- 🔧 **工具调用** — 实时显示 AI 使用的工具名称、参数和返回结果
- 📊 **Token 统计** — 会话头部显示当前上下文 token 使用量

## 切换模型

每个会话可以独立使用不同模型：

1. 点击会话头部的 **🤖 模型名称** 按钮
2. 从下拉列表选择新模型
3. 系统自动发送 `/model` 命令切换
4. 仅影响当前会话，其他会话不受影响

如果模型不在 Agent 白名单中，会显示 `Model "xxx" is not allowed` 错误。添加模型时 Dashboard 会自动同步白名单。

## 快捷命令

点击输入框旁的 **⚡** 按钮打开命令面板：

| 命令 | 功能 | 说明 |
|------|------|------|
| `/new` | 新建会话 | 在当前渠道创建新会话 |
| `/reset` | 重置上下文 | 清空对话历史，重新开始 |
| `/compact` | 压缩历史 | 将长对话压缩为摘要，节省 token |
| `/stop` | 停止运行 | 终止当前正在执行的任务 |
| `/model` | 查看模型 | 显示当前模型信息 |
| `/model xxx` | 切换模型 | 切换到指定模型 |
| `/think` | 推理级别 | 调整深度思考程度 |
| `/verbose` | 详细模式 | 显示更多执行细节 |
| `/fast` | 快速模式 | 减少延迟和 token 消耗 |
| `/agents` | 列出 Agent | 查看所有可用 Agent |
| `/sessions` | 列出会话 | 查看所有活跃会话 |

## 会话操作

| 按钮 | 功能 | 说明 |
|------|------|------|
| 📎 | 发送附件/图片 | 支持图片、文档等 |
| ↻ | 切换自动刷新 | 控制会话列表更新频率 |
| ＋ | 新建会话 | 创建新的对话 |
| ✏️ | 重命名 | 自定义会话显示名称 |
| ✕ | 删除会话 | ⚠️ 主会话不可删除 |

## 会话历史

- 每次点击会话时**自动刷新**最新历史记录
- 历史来自 Gateway 的 JSONL 存储文件
- 支持查看完整对话：用户消息 + AI 回复 + 工具调用 + 工具结果
- Token 使用量实时显示在会话头部

# ⏰ 定时任务

创建和管理自动化定时任务。

## 任务列表

- 🔍 **搜索** — 按名称关键词过滤
- 📊 **状态筛选** — 全部 / 正常 / 错误 / 已禁用
- ⏰ **状态图标** — ⏰ 运行中 / ❌ 有错误 / ⚫ 已禁用
- 📋 **计数** — 显示总数和启用数量

## 创建任务

1. 点击任务列表上方的「＋」按钮
2. 填写任务名称（如「天气播报」）
3. 设置调度方式（Cron 表达式或固定间隔）
4. 编写执行内容（Agent 消息提示词）
5. 可选：指定使用的模型
6. 启用任务

### Cron 表达式

| 表达式 | 含义 |
|--------|------|
| `*/15 * * * *` | 每 15 分钟 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 * * * *` | 每小时整点 |
| `0 */2 * * *` | 每 2 小时 |
| `0 */6 * * *` | 每 6 小时 |
| `0 8 * * *` | 每天 08:00 |
| `0 9 * * 1-5` | 工作日 09:00 |
| `0 12 * * *` | 每天中午 |
| `0 0 * * 0` | 每周日午夜 |
| `0 0 1 * *` | 每月 1 日 |

### 固定间隔

| 写法 | 含义 |
|------|------|
| `5m` | 每 5 分钟 |
| `30m` | 每 30 分钟 |
| `1h` | 每小时 |
| `2h` | 每 2 小时 |
| `1d` | 每天 |

## 任务操作

| 按钮 | 功能 |
|------|------|
| ▶ 手动执行 | 立即运行一次 |
| ✏️ 编辑 | 修改任务配置 |
| ⏸️ 启用/禁用 | 控制开关 |
| 🗑️ 删除 | 永久删除 |
| 📋 历史 | 查看执行日志 |

# 🧩 技能与插件

## 插件 (Plugins)

系统级插件，提供核心功能扩展：

# ❓ 常见问题

## Gateway 无法启动

1. 检查端口占用：`netstat -ano | findstr 18789`
2. 如被占用：`taskkill /PID <PID> /F`
3. 运行「🔍 环境状态 → 📋 诊断」
4. 检查 `~/.openclaw/openclaw.json` 语法
5. 尝试「🔄 重启」
6. 终极方案：删除 `~/.openclaw` 重新初始化

## WebSocket 连接失败

1. 确认 Gateway 已启动（状态栏应显示绿色）
2. 点击「🔐 添加WS权限」
3. 点击「🔓 启用WS验证」
4. 点击「🔌 WS测试」验证
5. 重启 Gateway

## 模型返回 "is not allowed"

原因：模型不在 Agent 白名单中。

解决：Dashboard 添加模型时会自动同步白名单。如果手动编辑配置，需在 `agents.defaults.models` 中添加 `"provider/model-id": {}`。

## 模型请求失败

1. 检查 API Key 是否正确且未过期
2. 确认 Base URL 可以访问
3. 使用「🧪 测试」验证连通性
4. 检查网络代理设置
5. 确认账户余额（付费模型）
6. 查看是否 429 限流

## 命令找不到（conda/python）

1. 确认工具已安装
2. 勾选「☐ Conda」激活环境
3. 或指定完整路径：`C:\ProgramData\anaconda3\python.exe`
4. 检查 PATH 环境变量

## 中文乱码

1. 点击「🌐 全局UTF-8」
2. 重启 Dashboard
3. 系统区域设置勾选「Beta: 使用 Unicode UTF-8」

## 启动闪退

1. 查看环境检测页面错误
2. 勾选 `--no-sandbox` 和 `--disable-gpu-sandbox`
3. 确认 Windows ≥ 10 64-bit
4. 更新显卡驱动
5. 尝试管理员身份运行

## 会话历史不显示

1. 等待 Gateway 完全启动
2. 刷新页面重新加载
3. 运行 `openclaw doctor --fix`

## 定时任务不执行

1. 确认任务状态为启用
2. 检查 Cron 表达式
3. 确认 Gateway 运行中
4. 尝试手动执行测试
