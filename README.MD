# 🚀 快速入门

## 系统要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | Windows 10 64-bit |
| 内存 | 4GB 以上 |
| Node.js | v18 或更高 |
| Git | 任意版本 |

## 首次启动

1. **双击** `OpenClaw Dashboard` 图标启动应用
2. 系统自动进入**环境检测**页面，逐一检查：
   - 🖥️ Windows 版本和 DirectX 11 支持
   - 💻 PowerShell 可用性和版本
   - 📦 Node.js 是否安装及版本号
   - 🔧 Git 是否可用
   - 🐍 Python 是否可用（可选组件）
   - 🔗 OpenClaw CLI 是否已安装
   - ⚙️ Gateway 配置文件是否存在且有效
3. 根据检测结果采取不同操作：
   - ✅ **全部通过** → 30 秒后自动进入主界面，或点击「进入APP」直接进入
   - ⚠️ **有警告项** → 可选择修复后进入，也可忽略直接进入
   - ❌ **致命错误** → 必须修复才能启动，点击「🔧 修复」按钮自动处理
4. 进入主界面后，点击顶部状态栏的 **▶ 启动** 按钮启动 Gateway 服务

## Gateway 管理

Gateway 是 OpenClaw 的核心服务进程，管理所有 AI 会话、渠道连接和模型调用。

| 操作 | 按钮 | 详细说明 |
|------|------|----------|
| 启动 | 「▶ 启动」 | 自动执行环境检测，通过后启动 Gateway 进程 |
| 停止 | 「■ 停止」 | 优雅停止 Gateway，断开所有 WebSocket 连接 |
| 重启 | 「🔄 重启」 | 自动执行 停止 → 等待 → 启动 流程 |
| 自启监控 | 「服务自启」复选框 | Gateway 异常退出时每 15 秒自动检测并重启 |

### 状态栏指示灯

| 状态显示 | 含义 | 需要操作 |
|----------|------|----------|
| 🟢 Gateway [已连接] | Gateway 正常运行，WS 连接成功 | 无需操作 |
| 🔴 Gateway [未连接] | Gateway 未启动或连接断开 | 点击「▶ 启动」 |
| 🟢 Session [在线] | 会话通道已建立，可正常对话 | 无需操作 |
| 🔴 Session [离线] | 会话通道未建立 | 等待 Gateway 完全启动 |
| 📊 Agents: N | 当前活跃 Agent 数量 | 参考信息 |
| 📊 Sessions: N | 当前活跃会话数 | 参考信息 |
| ⏱️ WS延迟 Nms | WebSocket 延迟 | >1000ms 表示网络不佳 |

# 🖥️ 控制台

控制台提供完整的 PowerShell 终端集成，可以直接在 Dashboard 中执行系统命令和查看 Gateway 日志。

## 终端功能

- ✅ **实时日志流** — Gateway 启动后的 stdout/stderr 输出实时显示
- ✅ **命令交互** — 支持直接输入和执行 PowerShell 命令
- ✅ **Conda 支持** — 勾选后自动激活 Conda base 环境
- ✅ **UTF-8 中文** — 自动设置编码，完美支持中文输出
- ✅ **高度可调** — 拖动输出区顶部蓝色条调整高度

## 命令输入

在终端下方的输入框中输入命令，按 **Enter** 或点击 **执行** 按钮运行。输入框执行的命令会在**独立结果显示区**中显示，不会干扰上方的终端日志流。

### 结果显示区操作

| 操作 | 方式 |
|------|------|
| 调整高度 | 拖动顶部蓝色条（向上拖增大） |
| 选中复制 | 鼠标选择文字后 Ctrl+C |
| 关闭 | 点击右上角 ✕ 按钮 |

## 快捷操作按钮

| 按钮 | 功能 |
|------|------|

| 🔐 添加 WS 权限 | 自动添加 gateway.controlUi.allowedOrigins |

# 🤖 模型管理

管理所有 AI 模型和 Provider（服务商）。

## 添加模型

### 方式一：快速添加（推荐）

1. 点击「＋ 添加模型」打开弹窗
2. 展开「⚡ 热门模型快速添加」区域
3. 浏览预设模型列表，点击「+ 添加」
4. 如果是新 Provider，会弹出输入框要求填写 API Key

### 方式二：手动添加

1. 点击「＋ 添加模型」
2. 选择已有 Provider 或输入新 Provider 名称（需同时填写 Base URL、API Key、API 类型）
3. 填写 Model ID 和显示名称
4. 点击「🧪 测试」验证后确认添加

### 重复检测

输入 Model ID 时实时检测：若已存在，输入框下方显示 ⚠️ 警告，点击添加时弹窗阻止。

## 设置主模型

| 按钮 | 作用 |
|------|------|
| 📝 主模型 | 设为文本对话主模型 |
| 🖼️ 主模型 | 设为图像/多模态主模型 |
| 📝 +备用 | 添加到文本备用列表 |
| 🖼️ +备用 | 添加到多模态备用列表 |

**自动故障转移**：主模型不可用时自动切换到备用模型。

# 💬 会话管理

管理所有 AI 对话会话，支持多会话并行。

## 发送消息

1. 从左侧列表**点击选择**一个会话
2. 底部输入框自动获得焦点
3. 输入消息，按 **Ctrl+Enter** 或点击「发送」
4. AI 回复以流式输出实时显示

### 快捷操作栏

| 按钮 | 功能 |
|------|------|
| 📎 | 发送附件/图片 |
| ↻ | 切换自动刷新 |
| 🤖 默认 | 切换当前会话模型 |
| ＋ | 新建会话 |
| ⚡ | 快捷命令面板 |

### 快捷命令

| 命令 | 功能 |
|------|------|
| `/new` | 新建会话 |
| `/reset` | 重置上下文 |
| `/compact` | 压缩历史 |
| `/stop` | 停止运行 |
| `/model xxx` | 切换模型 |

## 💭🔧📋 展开/收起开关

会话消息区标题栏右侧有三个独立开关按钮，用于控制消息内容的显示方式：

| 按钮 | 功能 | 默认状态 | 控制范围 |
|:-----|:-----|:---------|:---------|
| 💭 | 展开/收起思考过程 | 暗（收起） | AI 推理过程内容 |
| 🔧 | 展开/收起工具调用 | 亮（展开） | 工具调用卡片整体 + 参数 |
| 📋 | 展开/收起工具结果 | 暗（收起） | 所有 Output 内容（标题保留） |

### 使用方法

- **点击按钮**：切换该类内容的展开/收起状态
- **单击工具调用 header**：单独收起/展开某个工具调用卡片（不影响其他卡片）
- **状态同步**：切换按钮后，会话历史和实时流式气泡同步生效
- **会话保持**：切换会话时保持展开/收起状态

### 默认行为

- 新消息到达时：💭 收起、🔧 展开、📋 收起
- 思考过程默认隐藏，点击 💭 展开查看
- 工具调用卡片默认展开，点击 🔧 收起全部
- Output 内容默认收起（✅ Output 标题保留），点击 📋 展开查看

## 会话历史自动刷新

对话完成后（AI 回复结束或中断），Dashboard 自动在 2 秒后从 JSONL 文件重新加载完整会话历史，确保显示内容与存储一致。

## Token 统计

会话头部实时显示当前上下文 token 使用量，数据来自 Gateway 的 usage 统计。

## 创建任务

1. 点击任务列表上方的「＋」按钮
2. 填写任务名称
3. 设置调度方式（Cron 表达式或固定间隔）
4. 编写执行内容（Agent 消息提示词）
5. 可选指定模型
6. 启用任务

### Cron 表达式示例

| 表达式 | 含义 |
|--------|------|
| `*/15 * * * *` | 每 15 分钟 |
| `0 * * * *` | 每小时整点 |
| `0 */6 * * *` | 每 6 小时 |
| `0 8 * * *` | 每天 08:00 |
| `0 9 * * 1-5` | 工作日 09:00 |

### 固定间隔

| 写法 | 含义 |
|------|------|
| `5m` | 每 5 分钟 |
| `1h` | 每小时 |
| `1d` | 每天 |

# ❓ 常见问题

## Gateway 无法启动

1. 检查端口占用：`netstat -ano | findstr 18789`
2. 如被占用：`taskkill /PID <PID> /F`
3. 运行「🔍 环境状态 → 📋 诊断」
4. 检查 `~/.openclaw/openclaw.json` 语法
5. 尝试「🔄 重启」

## WebSocket 连接失败

1. 确认 Gateway 已启动
2. 点击「🔐 添加WS权限」→「🔓 启用WS验证」→「🔌 WS测试」
3. 重启 Gateway

## 模型返回 "is not allowed"

Dashboard 添加模型时会自动同步白名单。如果手动编辑配置，需在 `agents.defaults.models` 中添加 `"provider/model-id": {}`。

## 中文乱码

1. 点击「🌐 全局UTF-8」
2. 重启 Dashboard
3. 系统区域设置勾选「Beta: 使用 Unicode UTF-8」

## 会话历史不显示

1. 等待 Gateway 完全启动
2. 刷新页面重新加载
3. 运行 `openclaw doctor --fix`

## 思考过程不显示

AI 的思考过程需要模型支持（如 Claude Extended Thinking）。如果模型不支持，思考内容不会出现。支持的模型在回复时会在消息中包含 `thinking` 字段。

## 工具调用结果不显示

1. 确认 toolCallId 匹配
2. 内容超过 1500 字符会被截断，点击「展开全部」查看
3. 检查 📋 按钮是否已点亮（展开 Output）

# 🏗️ 会话系统架构（开发者）

## 数据存储路径

| 路径 | 内容 |
|------|------|
| `~/.openclaw/agents/main/sessions/sessions.json` | 会话索引 |
| `~/.openclaw/agents/main/sessions/<UUID>.jsonl` | 会话消息历史 |
| `~/.openclaw/cron/<UUID>.jsonl` | 定时任务执行历史 |

## 会话 Key 格式

| Key 格式 | 类型 |
|----------|------|
| `agent:main:main` | 🏠 主会话 |
| `agent:main:<channel>` | 渠道会话 |
| `agent:main:cron:<UUID>` | ⏰ 定时任务 |
| `agent:main:subagent:<UUID>` | 🔖 子任务 |

## 流式输出协议

AI 回复以流式方式推送：
```
chat event:
  state: "delta"   → 增量文本 + thinking 更新
  state: "final"   → 完整最终结果
  state: "aborted" → 中断

agent event (stream: "tool"):
  phase: "start"   → 工具调用开始
  phase: "update"  → 部分结果更新
  phase: "result"  → 最终结果
```

## 构建与打包

```powershell
# 验证语法
node -e "new Function(require('fs').readFileSync('renderer/app.js','utf8'))"
```
