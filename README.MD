# 🚀 快速入门

## 首次启动流程

1. **双击** `OpenClaw Dashboard` 图标启动
2. 系统自动进入**环境检测**页面，检查：
   - Windows 版本 / DirectX 11 / PowerShell / Node.js / Git
   - Python / OpenClaw CLI / Gateway 配置
3. 根据检测结果：
   - ✅ **全部通过** → 30 秒后自动进入（或点击「进入APP」）
   - ⚠️ **有警告** → 可选择修复后进入
   - ❌ **致命错误** → 必须修复才能启动
4. 进入主界面后，点击 **▶ 启动** 按钮启动 Gateway

## 停止 / 重启 Gateway

| 操作 | 方式 | 说明 |
|------|------|------|
| 启动 | 点击「▶ 启动」 | 自动执行 Gateway 检测 + 启动 |
| 停止 | 点击「■ 停止」 | 停止 Gateway，断开所有 WebSocket |
| 重启 | 点击「🔄 重启」 | 自动执行 停止→等待→启动 |
| 自启 | 勾选「服务自启」复选框 | Gateway 异常停止时每 15 秒自动重启 |

## 状态栏说明

状态栏位于顶部，显示实时连接状态：

| 状态 | 含义 |
|------|------|
| 🟢 Gateway [已连接] | Gateway 正常运行，WS 连接成功 |
| 🔴 Gateway [未连接] | Gateway 未启动或连接失败 |
| 🟢 Session [在线] | 会话通道已建立 |
| 🔴 Session [离线] | 会话通道断开 |

# 🖥️ 控制台

控制台提供完整的 **PowerShell 终端**，可直接操作 OpenClaw 系统。

## 终端功能

- ✅ 实时显示 Gateway 运行日志（stdout/stderr）
- ✅ 支持 PowerShell 命令交互
- ✅ 自动激活 Conda base 环境（勾选后）
- ✅ 支持中文显示（UTF-8 编码）

## 底部输入框

在终端下方的输入框中输入命令，按 **Enter** 或点击 **执行** 按钮执行。

```
PS> openclaw status
PS> openclaw doctor
PS> dir
PS> echo 你好
```

输入框执行的命令会在**独立结果显示区**中显示输出，不影响终端日志流。

## Conda 环境

输入框左侧有一个 **☐ Conda** 复选框：

| 状态 | 说明 |
|------|------|
| ☐ 未勾选（默认） | 命令直接通过 PowerShell 执行 |
| ☑ 已勾选 | 命令通过 `conda run -n base` 执行，自动激活 base 环境 |

> 💡 如果需要使用其他 Conda 环境，不勾选 Conda，手动在命令前加 `conda activate <env> &&`

## 结果显示区

结果显示区在输入框上方，支持：

- 📋 **可选中复制**（鼠标选择文字后 Ctrl+C）
- ✕ **可关闭**（右上角 ✕ 按钮）
- 🎨 **彩色显示**：绿色=命令行，白色=输出，红色=错误

## 快捷按钮

| 按钮 | 功能 | 说明 |
|------|------|------|
| 📁 自定义 OpenClaw 目录 | 创建 Junction 链接 | 将 `~/.openclaw` 重定向到自定义路径 |
| 🔐 添加 WS 权限 | 补全 WS 配置 | 自动添加 `gateway.controlUi.allowedOrigins` 等 |
| 🔓 启用 WS 验证 | 开启 WS 认证 | 设置 `allowInsecureAuth` |
| 🔌 WS 测试 | 测试连接 | 验证 WebSocket 连接是否正常 |

# 🤖 模型管理

管理所有 AI 模型和 Provider。

## 添加模型

```
步骤 1 → 点击「＋ 添加模型」
步骤 2 → 选择已有 Provider 或创建新 Provider
步骤 3 → 填写 Model ID 和显示名称
步骤 4 → 点击「🧪 测试」验证连通性
步骤 5 → 确认添加
```

## 设置主模型

每个模型卡片提供以下按钮：

| 按钮 | 作用 |
|------|------|
| 📝 主模型 | 设为文本对话主模型 |
| 🖼️ 主模型 | 设为图像/多模态主模型 |
| 📝 +备用 | 添加到文本备用模型列表 |
| 🖼️ +备用 | 添加到多模态备用模型列表 |

> ⚡ 主模型不可用时，系统自动切换到备用模型。

## Provider 管理

点击 Provider 卡片上的 **⚙ 编辑** 可修改配置：

| 字段 | 说明 |
|------|------|
| Base URL | API 地址（如 `https://api.openai.com/v1`） |
| API Key | 密钥 |
| 认证类型 | API Key / OAuth / Bearer Token |

点击 **🗑 删除** 可移除 Provider（连同所有模型）。

# ⏰ 定时任务

## 任务列表

左侧显示所有定时任务，支持：

- 🔍 **搜索**：按名称过滤
- 📊 **状态筛选**：全部 / 正常 / 错误 / 已禁用
- ⏰ **状态图标**：⏰ 运行中 / ❌ 错误 / ⚫ 已禁用

## 创建任务

```
步骤 1 → 点击任务列表上方的「＋」按钮
步骤 2 → 填写任务名称
步骤 3 → 设置调度（cron 表达式或间隔）
步骤 4 → 编写执行内容（Agent 消息）
步骤 5 → 启用任务
```

### Cron 表达式示例

| 表达式 | 含义 |
|--------|------|
| `*/30 * * * *` | 每 30 分钟 |
| `0 8 * * *` | 每天 08:00 |
| `0 */2 * * *` | 每 2 小时 |
| `0 9 * * 1-5` | 工作日 09:00 |

## 任务操作

- 🔄 **手动执行**：立即运行一次
- ✏️ **编辑**：修改配置
- ⏸️ **启用/禁用**：控制任务开关
- 🗑️ **删除**：永久删除任务
- 📋 **执行历史**：查看日志和输出

# 💬 会话管理

## 会话列表

左侧显示所有活跃会话：

- 📌 **主会话**：标记为主会话（不可删除）
- 🔍 **搜索**：按名称过滤
- 📊 **排序**：按最近更新时间

## 发送消息

```
步骤 1 → 从左侧选择会话
步骤 2 → 在底部输入框输入消息
步骤 3 → 按 Ctrl+Enter 或点击「发送」
```

## 切换模型

输入框左侧有**模型下拉框**：

1. 选择模型后自动发送 `/mode <provider>/<model>`
2. 该会话后续消息使用新模型

## 快捷命令

点击「快捷命令」按钮打开命令面板：

| 命令 | 功能 |
|------|------|
| `/new` | 新建会话 |
| `/reset` | 重置会话上下文 |
| `/compact` | 压缩历史（节省 token） |
| `/stop` | 停止当前运行 |
| `/model` | 查看/切换模型 |
| `/think` | 设置推理级别 |
| `/verbose` | 切换详细模式 |
| `/fast` | 切换快速模式 |
| `/mode <model>` | 切换当前会话模型 |

## 会话操作

| 按钮 | 功能 |
|------|------|
| 📎 | 发送附件/图片 |
| ↻ | 切换自动刷新模式 |
| ＋ | 新建会话 |
| ✏️ | 重命名会话 |
| ✕ | 删除会话（主会话除外） |

# ❓ 常见问题

## Q: Gateway 无法启动？

```
1. 检查端口 18789 是否被占用（netstat -ano | findstr 18789）
2. 运行「🔍 环境状态 → 📋 诊断」
3. 检查 ~/.openclaw/openclaw.json 配置语法
4. 尝试点击「🔄 重启」
```

## Q: WS 连接失败？

```
1. 确认 Gateway 已启动（状态栏显示绿色）
2. 点击「🔐 添加WS权限」补全配置
3. 重启 Gateway
```

## Q: 模型请求失败？

```
1. 检查 API Key 是否正确
2. 确认 Provider baseUrl 可达
3. 使用模型卡片的「🧪 测试」验证
4. 检查网络代理设置
```

## Q: Shell 命令找不到（如 conda）？

```
1. 确认 Conda 已安装
2. 勾选「☐ Conda」复选框
3. 如果使用自定义环境，手动执行：
   conda activate <env> && <你的命令>
```

## Q: 中文乱码？

```
1. 点击「🌐 全局UTF-8」设置代码页
2. 重启 Dashboard
3. 确认系统区域设置正确
```

## Q: 启动闪退？

```
1. 启动环境检测页面会自动检查依赖
2. 勾选「--no-sandbox」和「--disable-gpu-sandbox」
3. 确认 Windows 版本 ≥ 10
4. 更新显卡驱动
```
