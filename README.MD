# 🚀 快速入门

## 系统要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | Windows 10 64-bit |
| 内存 | 4GB 以上 |
| Node.js | v18 或更高 |
| Git | 任意版本 |

## 首次启动

1. **双击** `OpenClaw Dashboard` 图标启动应用
2. 系统自动进入**环境检测**页面，逐一检查：
   - 🖥️ Windows 版本和 DirectX 11 支持
   - 💻 PowerShell 可用性和版本
   - 📦 Node.js 是否安装及版本号
   - 🔧 Git 是否可用
   - 🐍 Python 是否可用（可选组件）
   - 🔗 OpenClaw CLI 是否已安装
   - ⚙️ Gateway 配置文件是否存在且有效
3. 根据检测结果采取不同操作：
   - ✅ **全部通过** → 30 秒后自动进入主界面，或点击「进入APP」直接进入
   - ⚠️ **有警告项** → 可选择修复后进入，也可忽略直接进入
   - ❌ **致命错误** → 必须修复才能启动，点击「🔧 修复」按钮自动处理
4. 进入主界面后，点击顶部状态栏的 **▶ 启动** 按钮启动 Gateway 服务

# 🖥️ 军机处总控台

Dashboard 的核心控制面板，提供 Gateway 服务管理、系统监控和命令行终端。

## 顶部状态栏

显示 Gateway 运行状态、WebSocket 连接信息和系统时钟。

| 状态显示 | 含义 | 需要操作 |
|----------|------|----------|
| 🟢 Gateway [已连接] | Gateway 正常运行，WS 连接成功 | 无需操作 |
| 🔴 Gateway [未连接] | Gateway 未启动或连接断开 | 点击「▶ 启动」 |
| 🟢 Session [在线] | 会话通道已建立 | 无需操作 |
| 🔴 Session [离线] | 会话通道未建立 | 等待 Gateway 启动 |
| 📊 Agents: N | 当前活跃 Agent 数量 | 参考信息 |
| 📊 Sessions: N | 当前活跃会话数 | 参考信息 |
| ⏱️ WS延迟 Nms | WebSocket 延迟 | >1000ms 表示网络不佳 |

### 操作按钮

| 按钮 | 功能 |
|------|------|
| ▶ 启动 | 启动 Gateway 服务 |
| ■ 停止 | 优雅停止 Gateway |
| 🔄 重启 | 停止 → 等待 → 启动 |
| ☑ 服务自启 | Gateway 异常退出时自动重启 |

# 🏛️ 三省六部

基于古代帝国制度设计的 AI 多 Agent 协作系统，由 12 个 AI Agent 组成。

## 架构概览

```
用户消息 → 👑 太子(消息分拣) → 📜 中书省(规划) → 🔍 门下省(审核) → 📜 尚书省(派发)
                                                          ↓
                                          六部并行执行：💰户部 📝礼部 ⚔️兵部 ⚖️刑部 🔧工部 📋吏部
```

## 前置要求

| 组件 | 要求 |
|------|------|
| Git | 任意版本 |
| Python | 3.10+（推荐 Anaconda） |
| Node.js | 18+ |
| OpenClaw | 已运行中 |

## 安装方式

### 方式一：一键安装（推荐）

1. 点击「🏛️ 三省六部」标签
2. 点击「🚀 一键安装三省六部」按钮
3. 等待 2-5 分钟，自动完成：克隆仓库 → 安装依赖 → 注册 11 个 Agent

### 方式二：手动安装

1. 克隆仓库：`git clone https://ghfast.top/https://github.com/cft0808/edict.git ~/edict`
2. 安装依赖：`cd ~/edict && powershell -ExecutionPolicy Bypass -File .\install.ps1`
3. 启动数据刷新：在 Git Bash 中运行 `cd ~/edict && bash scripts/run_loop.sh`
4. 启动看板：`cd ~/edict && python dashboard\server.py`
5. 访问：`http://127.0.0.1:7891`

## 运行管理

| 按钮 | 功能 |
|------|------|
| 🚀 一键启动全部 | 启动数据刷新循环 + 看板服务器 |
| ■ 停止全部 | 停止所有后台进程 |
| 🔄 启动数据刷新 | 启动定时数据同步（后台运行） |
| 🌐 启动看板服务器 | 启动 Flask 看板（端口 7891） |
| 🔗 打开看板 | 在嵌入窗口中打开看板 |
| 🌐 浏览器打开 | 在系统浏览器中打开看板 |
| 🔧 重装/修复 | 重新运行安装脚本 |

# 💬 会话管理

管理所有 AI 对话会话，支持多会话并行。

## 会话列表

左侧显示所有会话：

- 📌 **主会话** — 标记为主会话，不可删除
- 🔍 **搜索过滤** — 输入关键词过滤会话名称
- 📊 **筛选** — 全部 / 🟢 活跃 / ⚪ 空闲

2. 底部输入框自动获得焦点
3. 输入消息，按 **Ctrl+Enter** 或点击「发送」

# ⏰ 定时任务

创建和管理自动化定时任务。

## 创建任务

1. 点击任务列表上方的「＋」按钮
2. 填写任务名称
3. 设置调度方式（Cron 表达式或固定间隔）
4. 编写执行内容（Agent 消息提示词）
5. 可选指定模型
6. 启用任务

### Cron 表达式示例

| 表达式 | 含义 |
|--------|------|
| `*/15 * * * *` | 每 15 分钟 |
| `0 * * * *` | 每小时整点 |
| `0 */6 * * *` | 每 6 小时 |
| `0 8 * * *` | 每天 08:00 |
| `0 9 * * 1-5` | 工作日 09:00 |

### 固定间隔

| 写法 | 含义 |
|------|------|
| `5m` | 每 5 分钟 |
| `1h` | 每小时 |
| `1d` | 每天 |

## 任务操作

| 按钮 | 功能 |
|------|------|
| ▶ 手动执行 | 立即运行一次 |
| ✏️ 编辑 | 修改任务配置 |
| ⏸️ 启用/禁用 | 控制开关 |
| 🗑️ 删除 | 永久删除 |
| 📋 历史 | 查看执行日志 |

# 🤖 模型管理

管理所有 AI 模型和 Provider（服务商）。

## 添加模型

### 方式一：快速添加（推荐）

1. 点击「＋ 添加模型」打开弹窗
2. 展开「⚡ 热门模型快速添加」区域
3. 浏览预设模型列表，点击「+ 添加」
4. 如果是新 Provider，会弹出输入框要求填写 API Key

### 方式二：手动添加

1. 点击「＋ 添加模型」
2. 选择已有 Provider 或输入新 Provider 名称（需同时填写 Base URL、API Key、API 类型）
3. 填写 Model ID 和显示名称
4. 点击「🧪 测试」验证后确认添加

### 重复检测

输入 Model ID 时实时检测：若已存在，输入框下方显示 ⚠️ 警告，点击添加时弹窗阻止。

## 设置主模型

| 按钮 | 作用 |
|------|------|
| 📝 主模型 | 设为文本对话主模型 |
| 🖼️ 主模型 | 设为图像/多模态主模型 |
| 📝 +备用 | 添加到文本备用列表 |
| 🖼️ +备用 | 添加到多模态备用列表 |

**自动故障转移**：主模型不可用时自动切换到备用模型。

# ❓ 常见问题

## Gateway 无法启动

1. 检查端口占用：`netstat -ano | findstr 18789`
2. 如被占用：`taskkill /PID <PID> /F`
3. 运行「🔍 环境状态 → 📋 诊断」
4. 检查 `~/.openclaw/openclaw.json` 语法
5. 尝试「🔄 重启」

## WebSocket 连接失败

1. 确认 Gateway 已启动
2. 点击「🔐 添加WS权限」→「🔓 启用WS验证」→「🔌 WS测试」
3. 重启 Gateway

## 状态卡片显示"未检测"

- 确认对应服务已安装或已启动
- Dashboard 启动后自动检测，每 15 秒刷新一次
- 如果手动启动服务，等待 15 秒后自动同步状态

## 像素办公室标签空白

- 手动启动办公室后，切换标签时自动重新加载
- 如果仍空白，点击「🌐 进入小江的像素办公室」按钮刷新

## 悬浮窗打不开

- 悬浮窗关闭后需要重新点击按钮创建
- 同一个悬浮窗重复点击会显示已存在的窗口
- 如果窗口被遮挡，尝试按 Esc 关闭其他弹窗

## 模型返回 "is not allowed"

Dashboard 添加模型时会自动同步白名单。如果手动编辑配置，需在 `agents.defaults.models` 中添加 `"provider/model-id": {}`。

## 中文乱码

1. 点击「🌐 全局UTF-8」
2. 重启 Dashboard
3. 系统区域设置勾选「Beta: 使用 Unicode UTF-8」

## 会话历史不显示

1. 等待 Gateway 完全启动
2. 刷新页面重新加载
3. 运行 `openclaw doctor --fix`

## 思考过程不显示

AI 的思考过程需要模型支持（如 Claude Extended Thinking）。支持的模型在回复时会在消息中包含 `thinking` 字段。

## 流式输出协议

AI 回复以流式方式推送：
```
chat event:
  state: "delta"   → 增量文本 + thinking 更新
  state: "final"   → 完整最终结果
  state: "aborted" → 中断

agent event (stream: "tool"):
  phase: "start"   → 工具调用开始
  phase: "update"  → 部分结果更新
  phase: "result"  → 最终结果
```
